# Session Summaries

Automatic Wiki-style documentation of Claude Code sessions. Each session is summarized and indexed for future reference.

## How it works

1. **Automatic capture** — At the end of each session, metadata is captured (branch, commit, timestamp)
2. **Manual summary** — Claude generates a detailed summary with accomplishments, decisions, and next steps
3. **Indexing** — Each summary is added to `INDEX.md` for easy browsing
4. **Searchable** — Organized by date, linked by topic

## Summary Structure

Each session file includes:
- **Date & time** — when the session occurred
- **Branch** — which branch was active
- **Accomplishments** — what was done (commits, files changed)
- **Decisions** — key technical or process decisions made
- **Blockers** — any obstacles encountered
- **Next steps** — what to do in the next session

## Accessing summaries

- Browse `INDEX.md` for an overview of all sessions
- Read individual `.md` files for detailed session notes
- Reference decisions and patterns from previous work
